其他的early decision predicates

Nov 6, 2023

教科书链接：https://link.springer.com/book/10.1007/978-3-319-94141-7

  • 之前的early stopping 算法中用到的predicate是：

    nbri[r] - nbri[r - 1] = 0

    我们这里称之为DIFF(i, r)

  • 不妨令 faultyi[r] = n - nbri[r]，即在第r轮，pi认为挂掉的process的数量。

  • 则 COUNT(i,r) ≡ (faultyi[r] ≤ r) 也可以是一个正确的early decision predicate

    因为：

    当COUNT(i,r)被满足时，现在的轮数大于pi挂掉的process的个数，

    也即在pi看来，存在这么一轮r’，1 ≤ r’ ≤ r，在该轮中没有process 挂掉。

    那在该r’轮结束时，pi里的viewi已经有当时能够知道的所有<k,v>元组并且由于不存在recover，以后也不会知道更多的<k,v>.

  • DIFF(i,r) 和 COUNT(i,r) 孰优孰劣？

    可以证明：

    • 给定一个execution，不妨令 r ≥ 2是满足COUNT(i,r)的第一轮。则存在COUNT(i,r) ⇒ DIFF(i,r)

    • 给定一个execution，不妨令 r ≥ 2是满足DIFF(i,r)的第一轮。则存在failure pattern，使得DIFF(i,r) ∧ ¬ COUNT(i,r)。

  • 第一条是因为当COUNT(i,r)满足时，也就是说foultyi[r] < r 并且 faultyi[r - 1] ≥ r - 1。

    则有faultyi[r] - faultyi[r - 1] < 1

    也即nbri[r - 1] - nbri[r] < 1

    又因为nbri[r - 1] - nbri[r] ≥ 0

    所以 nbri[r - 1] - nbri[r] = 0。

    也就是说COUNT(i,r) ⇒ DIFF(i,r)

  • 第二条可以反证：

    不妨令在某次execution中一开始就有2 ≤ x ≤ t个process 挂掉，之后没有一个process挂掉。

    则COUNT(i,r)在第x + 1轮变为真。

    而从第2轮起，nbri[r - 1]和nbri[r]之间的差即为0.

    所以对于2到x中的任意轮，DIFF(i,r) ∧ ¬ COUNT(i,r)

  • 上面两个定理说明：DIFF(i,r)优于COUNT(i,r)。

    因为存在某些failure pattern使得用DIFF(i,r)的算法可以在min(f + 2, t + 1)轮之前就终止。

    其根本原因是DIFF(i,r)可以把本轮中挂掉的process数也加入到考量中。

  • 从CSMPn,t[∅]上的early stopping的interactive consistency算法可以很直接得到early stopping 的consensus算法。

    因为是consensus，所以不需要view，只需要一个est.

    此处略过不表。
