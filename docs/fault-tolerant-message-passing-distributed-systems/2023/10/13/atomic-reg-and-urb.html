atomic寄存器与urb广播

Oct 13, 2023

教科书链接：https://link.springer.com/book/10.1007/978-3-319-94141-7

  • 如果D是一个failure detector class，并且在CAMPn,t[D]里可以构造出atomic register，不妨令A是该构造atomic register的算法，

    则总可以用A构造出算法，模拟出Σ,使得每个process pi得到simgaiτ

    结论：只要有failure detector class Σ，我们就可以在一个任意process可能crash的异步系统里构造出atomic寄存器

  • 对比failure detector class Θ 和 Σ

    • 只要有Θ，在CAMPn,t[-FC, t ≥ n/2]里就可以构造出URB 广播

    • 只要有Σ，就可以在CAMP里构造出atomic 寄存器

    • 并且两者的实现代码相同，那是否两者的computation power也一样呢？

    结论：在任何t ≥ n/2的系统里，Σ严格地强于Θ。即Θ可以在CAMPn,t[Σ]里被构造，而Σ无法在CAMPn,t[Θ]里被构造出来

    • 推理简述：

      回忆两者的properties，它们的liveness都是一样的，区别在于

      • Σ 上要求 intersection: 在任何时刻，任意两个process读到的sigma值总有交集 （可以prevent cluster partitioning）

      • Θ 上要求correctness：在任何时间，任何trustedi至少包含一个正常运行的process。

      • 当t < n/2时，从Σliveness和intersection结合起来看，可以得出Θ的correctness。

      • 当t < n/2时，却可以通过反证法得到Σ无法在CAMPn,t[Θ]里被构造出来。

  • 可以用atomic寄存器构造出URB广播

    构造算法简述

    • 算法由一组REG[i..n]组成，每个REG[i]都是一个SWMR的atomic register。每个REG[i]都允许pi写，允许所有process读。

    • 每个pi的本地变量有：

      • senti

      • regi[1..n]

      这些local variable里都放着一串message

    • urb-broadcast:

      首先把msg concatenate到senti。

      然后pi调用REG[i].write(senti).

    • urb-deliver:

      urb-deliver由一个background task实现：

      这个task是一个while true loop：

      在loop里面pi会从每个REG[x]里读取msg，然后放到本地regi[x]里。对于regi[x]里每条没有被urb-deliver的msg，去urb-deliver该msg。

  • 从上面结论中我们可以看到，无论t的值，Θ都可以在CAMP[Σ]或者CAMP[-FC, Σ]里被构造，而反过来不行。

    同时Σ又是满足可以被加到CAMP[∅]中后能无论t的值都能实现atomic寄存器的the weakest failure detector class。

    这就意味着从failure detector class的角度来看，atomic寄存器strictly stronger than URB广播
